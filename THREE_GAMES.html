Shared Narrative Structure

All three systems target the same output: 15 Save the Cat beats organized into 4 acts (Act 1, Act 2A, Act 2B, Act 3) with 8 syntagma codes (AS, PS, BS, DS, ALT, SC, ES, OS).

Source: narrative-garden-v03.html — Reference implementation with visual act tracking and syntagma selection.

1. Matrix Editor — Defense as Editorial

const game = {
  chains: [],        // Collected sequences
  threats: [],       // Incoming fragments
  systems: [],       // Defensive structures
  pendingChain: {    // Chain awaiting assignment
    lane: 0,
    fragments: [...], // 3+ clips
    threats: [...]
  }
}

// Collection trigger:
if (laneFragments.length >= 3) {
  game.pendingChain = { lane, fragments };
  showBeatSyntagmaSelector();
}

// Storage after assignment:
game.chains.push({
  beatName: "Opening Image",
  beatNumber: 1,
  syntagmaType: "DS",
  fragments: [...],  // Actual video clips with timecodes
  lane: 0,
  wall: {...}
});

Key insight: Fragments are hostile "threats" advancing in lanes. You don't move fragments—you deploy systems and let them collide. Collection is passive capture, not active arrangement. Storage preserves lane context and wall metadata.

2. Garden — Tetris as Montage

// 2D spatial grid
for(let y=0; y<H; y++) {
  grid[y] = Array(W).fill(null);
}

// Falling block system
const piece = {
  type: 'L',     // Tetromino shape
  x: 5,
  y: 0,
  rotation: 0,
  fragmentId: 'clip_001'  // Link to video
};

// Lock when landing
if (canPlace(piece, grid)) {
  lockPiece(piece);
  checkLines();
}

// Clear horizontal lines → lock sequence
if (isLineComplete(y)) {
  const fragments = grid[y].filter(cell => cell);
  lockedSequences.push({
    fragments: fragments,
    level: recursiveLevel
  });
}

Key insight: Fragments are blocks falling under gravity. You actively rotate and place them. The grid is spatial memory—position matters. Clearing lines converts spatial arrangement into temporal sequence. Has recursive timeline (48→24→12→6→3→1 layers).

3. Flux Garden — Emergence as Assembly

  sequence: [],       // Collected matches
  flowField: [],      // Perlin noise vectors
  collectedTypes: new Set(),
  typeSegmentVideos: {}  // Video segments by type
};

// Match detection
function findMatches() {
  // Horizontal + vertical 3+ matches
  return matches;
}

// Collection
if (matches.length > 0) {
  game.sequence.push({
    type: cellType,
    count: matches.length,
    timestamp: Date.now()
  });
  
  // Play corresponding video segment

Key insight: Hybrid system. Match-3 mechanics drive fragment collection, but flow field provides ambient visualization. Storage is minimal—just type counts and sequences. More about procedural generation than manual assembly. Lacks explicit beat/syntagma assignment.

Storage Philosophy

System Storage Model Metaphor
Matrix Editor Timeline-centric (lanes as timelines) Horizontal accumulation
Garden Spatial-centric (2D grid coordinates) Vertical stacking + horizontal clearing
Flux Garden Type-centric (collect by symbol type) Pattern matching

Export Capability

Matrix Editor: Full 5-file export system (OTI JSON, EDL, Storymap.md, Shotlist.js, Chart.d3.js). Complete backend for film generation.

Garden: Partial export (EDL, basic formats). Needs OTI JSON + full export parity.

Flux Garden: No export currently. Needs OTI integration + storymap generation.

Unification Strategy

Use oti-export.js universal library to add export capability to all three systems:

  • Matrix Editor: OTIExport.fromMatrixEditor(game)
  • Garden: OTIExport.fromGardenEditor(timeline)
  • Flux Garden: OTIExport.fromMatch3Sequencer(game.sequence)

All three export to same OTI JSON format → all three can feed into prompt-lab.html film generator.
